# Proyecto

Este repo mezcla dos partes relacionadas:
1. Un backend FastAPI
2. Un bot de Telegram

# Objetivo de hoy

Ordenar el proyecto sin reescribirlo desde cero.
Quiero una refactorización mínima, segura y gradual.

# Prioridades

1. Entender la estructura actual
2. Separar responsabilidades mejor
3. Evitar archivos duplicados o ambiguos
4. Mantener el bot funcionando
5. Dejar una forma clara de ejecutar backend y bot

# Reglas

- No hacer una reescritura total
- No mover archivos innecesariamente
- Cambios pequeños y verificables
- Explicar cada cambio antes de aplicarlo
- Mantener compatibilidad con lo actual
- Mejorar organización, imports, config y logging

# Qué no hacer

- No agregar frameworks nuevos
- No complicar la arquitectura
- No inventar funcionalidades nuevas
